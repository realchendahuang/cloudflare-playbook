## 先跳过

| 能力 | 为什么先跳过 |
| --- | --- |
| 浏览器侧网络失败上报 | 只能补充用户侧网络诊断，不替代 Workers Logs、Web Analytics 或业务日志。 |
| 公开可验证随机数 | 不能拿来生成密钥、访问凭证或会话密钥。 |
| 代管客户 Cloudflare 账号 | 面向合作伙伴场景，早期 SaaS 先看普通多租户。 |
| 企业配置版本管理 | 适合企业域名配置灰度，不适合早期项目。 |
| 隐私代理基础设施 | Access、Tunnel、WAF 仍然按各自场景使用。 |
| 新媒体协议实验 | 视频先看 Stream，实时音视频先看 Realtime。 |
| 助手长期记忆 | 没有明确记忆需求时，先用 D1 / R2 / Vectorize。 |

## 可以现在看

| 场景 | 看什么 | 判断 |
| --- | --- | --- |
| Cloudflare 资源变多，想按环境、团队、成本中心归类。 | 资源标签 | 能做治理工具，但不用在第一个 Worker 上引入。 |
| 用户反馈“打不开”，但服务端没有请求记录。 | 网络失败上报 | 只用于补充诊断用户侧网络问题。 |
| 做公开抽签或协议随机数，需要可验证随机源。 | 公开随机源 | 只用于公开可验证流程，不用于私密凭证。 |

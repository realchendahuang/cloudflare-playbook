---
title: 流量调度
---

## 适用场景

单站点、单源站先用代理、缓存、WAF 和源站防火墙。有主备源站或多区域源站时，再看负载均衡和健康检查。只是想知道源站是否在线，可以先做健康检查和告警。

要公开非 HTTP 服务，看四层入口产品。全球回源慢且缓存已做好时，再看路径优化。私有服务有多个入口时，先做私网方案设计。

## 推荐顺序

1. 先保证所有网站流量真的经过 Cloudflare。
2. 单源站阶段只做基础监控和源站保护。
3. 有主备、多区域或蓝绿发布场景后，再做自动切换。
4. 非 HTTP 服务、私有网络和专线场景，放到企业网络阶段评估。
5. 缓存和源站性能都做好后，再判断路径优化是否值得付费。

## 产品分工

负载均衡在多源站故障切换时才有价值。健康检查只负责发现问题，自动切流还要配合负载均衡策略。四层入口只给非 HTTP 公开服务用。路径优化只优化回源路径，不替代缓存和源站性能优化。私网调度放到私有网络、多数据中心和企业网络阶段再看。

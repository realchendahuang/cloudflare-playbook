## 一句话判断

免费计划包含 DDoS 防护。普通项目优先把入口接正确，比研究高级规则更有用。Workers Paid 不会增强 DDoS 防护，它只影响开发者平台额度。

域名接入 Cloudflare 后不等于自动安全。未代理记录、源站直连和泄露 IP 都可能绕过 Cloudflare。DDoS 也不能解决评论垃圾内容，写入口要配合 WAF、限流、Turnstile 和业务权限。

## 默认顺序

1. 公开网站记录保持代理开启。
2. 源站只接受 Cloudflare 回源和可信运维入口。
3. 静态资源放到 Static Assets、Pages、R2 或 Cache。
4. DDoS 防护保持默认。
5. 写入口加 WAF、限流和 Turnstile。
6. 开基础告警和账单提醒。

## 源站保护

网站记录开启代理，让请求经过 Cloudflare 防护。审计未代理记录，旧子域、邮件、监控和配置都可能泄露源站 IP。邮件和网站源站尽量分离，因为 MX、退信和邮件日志会暴露服务器地址。

源站防火墙要收窄入口，只允许 Cloudflare IP 段、可信运维入口或 Tunnel。旧 IP 已经泄露时，攻击后要考虑更换源站 IP，单纯接入 Cloudflare 不足。后台走 Tunnel / Access，管理入口通过 Access 或 Tunnel 进入。

## DDoS、WAF、限流的分工

| 能力 | 解决范围 |
| --- | --- |
| DDoS 防护 | 大规模异常流量、连接压力和协议层攻击。 |
| 缓存 / CDN | 减少攻击或突发流量对源站的回源压力。 |
| WAF | 已知漏洞、危险路径和明确业务入口。 |
| 限流 | 登录、评论、搜索、上传和写接口被刷。 |
| Turnstile | 需要人机判断的表单和写入口。 |

## 临时防护模式慎用

Under Attack Mode 适合正在被打且无法定位路径时短时间使用。更推荐按路径开启，用在登录、后台、写接口或被打路径。不要长期全站开启，攻击结束后回到 WAF 和限流的精确规则。它也不能替代源站保护，源站直连问题要单独处理。

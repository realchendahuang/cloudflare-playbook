## 适用场景

少量站点配图跟随静态站发布，重点控制尺寸。用户上传图片或附件时，原文件进 R2，业务索引进 D1。同一张图需要多尺寸时，先固定少量缩略图尺寸，再看 Images。

视频是产品体验的一部分时，看 Stream 的播放、签名、分析和录制能力。页面变慢时，先查图片尺寸、缓存率、第三方脚本和线上访问数据。第三方脚本变多时，能删就删，必要脚本再交给 Zaraz 管理。截图、PDF、动态抓取看 Browser Run 的浏览器时长和并发。

## 成本指标

R2 看存储量、读写操作和公开下载热点。Images 看转换参数数量、原图大小和常用尺寸缓存。Stream 看存储分钟数、播放分钟数和私有内容访问控制。Zaraz 看第三方工具真正需要的事件量。Browser Run 看浏览器时长、并发，以及是否及时关闭浏览器会话。

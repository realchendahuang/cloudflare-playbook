## 适用场景

| 问题场景 | 优先处理方式 |
| --- | --- |
| 少量站点配图 | 跟随静态站发布，控制尺寸。 |
| 用户上传图片或附件 | 原文件进 R2，业务索引进 D1。 |
| 同一张图需要多尺寸 | 固定少量缩略图尺寸，再看 Images。 |
| 视频是产品体验的一部分 | 看 Stream 的播放、签名、分析和录制能力。 |
| 页面变慢 | 查图片尺寸、缓存率、第三方脚本和线上访问数据。 |
| 第三方脚本变多 | 删脚本，必要脚本再交给 Zaraz 管理。 |
| 截图、PDF、动态抓取 | 看 Browser Run 的浏览器时长和并发。 |

## 成本指标

| 产品 | 重点指标 |
| --- | --- |
| R2 | 存储量、读写操作、公开下载热点。 |
| Images | 转换参数数量、原图大小、常用尺寸缓存。 |
| Stream | 存储分钟数、播放分钟数、私有内容访问控制。 |
| Zaraz | 第三方工具真正需要的事件量。 |
| Browser Run | 浏览器时长、并发、是否及时关闭浏览器会话。 |

最后核对日期：2026-06-18。

R2 是 Cloudflare 的 S3 兼容对象存储。它适合放文件、图片、附件、备份、导入导出结果、静态资源原始文件和数据湖对象。

先记住一句：**R2 适合存对象，不适合当数据库，也不适合塞进前端构建产物里。**

## 该不该用

| 场景 | 判断 |
| --- | --- |
| 用户上传图片、附件、导出文件 | 优先用。文件本体进 R2，数据库只存 key 和 metadata。 |
| 公开下载、软件包、PDF、备份 | 适合。注意 Class B 读取次数。 |
| 原图存储 + 缩略图处理 | 适合。原图进 R2，转换看 Images 或 Worker。 |
| D1 里想存大 JSON、图片、二进制 | 不适合。D1 存索引和状态，R2 存对象。 |
| 前端构建产物 | 不优先。Workers Static Assets / Pages 更合适。 |
| 视频产品 | 不优先。大规模播放看 Stream，普通文件下载再看 R2。 |
| 高频小对象读写 | 谨慎。操作次数会计费，缓存策略要先设计。 |

## 免费与付费边界

| 能力 | Free | 付费边界 | 判断 |
| --- | --- | --- | --- |
| Standard storage | 10 GB-month / month | $0.015 / GB-month | 小项目文件、图片、备份很够用。 |
| Class A operations | 1M / month | $4.50 / million | 写入、列举、创建 multipart 等偏修改操作。 |
| Class B operations | 10M / month | $0.36 / million | 读取、下载、head 等偏读取操作。 |
| Egress | 免费 | 免费 | 这是 R2 最大优势，但不代表 R2 完全免费。 |
| Infrequent Access | 无免费额度 | 存储更便宜，读取和取回另计 | 冷数据再用，不要一开始默认上。 |

R2 的成本主要看三件事：存了多少、写了多少、读了多少。公开下载场景最容易低估 Class B；批量上传和生命周期处理最容易低估 Class A。

## Class A / B 怎么看

| 类型 | 常见动作 | 成本直觉 |
| --- | --- | --- |
| Class A | put、copy、post、list、multipart upload、生命周期转换 | 改变状态或枚举对象，通常更贵。 |
| Class B | get、head、下载对象 | 读取对象，单价低但量可能很大。 |

判断成本时，不要只看存储容量。一个 1 GB 的冷备份可能很便宜，一个很小但每天被大量下载的文件也可能让 Class B 增长很快。

## 设计原则

| 原则 | 判断 |
| --- | --- |
| 数据库只存引用 | D1 存 object key、owner、size、content-type、状态和权限。 |
| 上传要做权限边界 | 用户上传不要直接裸写公共 bucket。 |
| 下载要区分公开和私有 | 公开文件可走自定义域名；私有文件用签名 URL 或 Worker 鉴权。 |
| 大文件不要进 Worker 内存 | Worker 做鉴权和路由，尽量流式处理。 |
| 热点读要加缓存 | 公开下载和图片访问要考虑 CDN cache。 |
| 冷数据再考虑 IA | Infrequent Access 适合低频访问，不适合热文件。 |

## 和其他产品怎么选

| 需求 | 优先产品 |
| --- | --- |
| 静态站构建产物 | Workers Static Assets / Pages |
| 用户上传和下载文件 | R2 |
| 关系数据、评论、订单、权限 | D1 |
| 读多写少配置和缓存 | KV |
| 图片裁剪、格式转换、响应式图 | Images + R2 原图 |
| 视频播放、转码、播放器 | Stream |
| 数据湖和离线查询 | R2 + R2 Data Catalog / R2 SQL |

## 常见误区

| 误区 | 更好的做法 |
| --- | --- |
| R2 免 egress，所以完全免费 | 存储、Class A、Class B 仍然会计费。 |
| 把 R2 当数据库 | R2 存对象，查询和事务交给 D1。 |
| 把用户文件放进 Static Assets | 用户文件进 R2，部署产物才进 Static Assets。 |
| 所有文件都公开 | 私有文件走 Worker 鉴权或签名 URL。 |
| 一开始就用 Infrequent Access | 先确认访问频率；热文件用 Standard。 |
| 大量 list 对象做业务查询 | 把索引写到 D1，不要靠 list 扫 bucket。 |

## 先别看产品，先看免费额度

最后核对日期：2026-06-18。

Cloudflare 的价格、额度和计划边界会变化。本站数字以官方价格和限制页面为最终依据；首页先把起步项目最该看的免费额度摊开，完整口径见 [免费额度大全](/platform/free-paid/)。

这个 Playbook 不从产品名开始，而是从免费额度开始。免费额度不是附录，也不是试用装；它是一套可以组合的起步架构。先回答四个问题：

- 项目能不能先 0 元跑起来。
- 0 元能跑到什么程度。
- 跑到哪里才该付费。
- 付的那 5 美元到底买到什么。

| 先问 | 直接答案 |
| --- | --- |
| 文档站、官网、博客能不能 0 元跑？ | 可以。静态资产请求免费且不限量，先用 Workers Static Assets / Pages。 |
| 小接口、评论、表单能不能 0 元验证？ | 可以。Workers Free、D1、KV、R2、Queues、Durable Objects 都有免费层。 |
| 免费阶段最该防什么？ | 防公开写入口被刷、防静态流量误进 Worker、防数据库全表扫描、防 AI 调用不设限。 |
| 什么时候付 5 美元/月？ | 动态请求、CPU、日志、队列、D1/KV/DO 等开发者平台能力进入真实生产后再付。 |

## All the fun：0 元起步配方

普通项目最先需要的不是“买哪个计划”，而是把静态内容、动态入口、数据、文件、安全和搜索放到正确位置。0 元不是只能放静态页，真正好用的地方在于把这些免费层组合起来：

| 项目 | 0 元组合 | 先看哪条线 |
| --- | --- | --- |
| 开源文档站 | Workers Static Assets / Pages + Pagefind + Web Analytics | 静态请求免费且不限量；Pages 每月 500 次构建。 |
| 教程站轻社区 | 静态资产层 + Twikoo Cloudflare + D1 + Turnstile | D1 每天 100k 行写入；公开入口要防刷。 |
| 小接口 / 第三方回调 | Workers Free + D1 / KV + 最小 WAF / 限流 | 100,000 次动态请求/天；10 ms CPU/次调用。 |
| 表单 / 反馈箱 | Workers Free + D1 + Turnstile + Workers Logs | D1 写入、日志 3 天留存、垃圾提交。 |
| 上传下载工具 | R2 + 签名链接 + CDN 缓存 | 每月 10 GB 标准存储；1M 写入类和 10M 读取类操作。 |
| 小团队后台 | Access + Tunnel + D1 | Zero Trust Free 50 users；后台不要裸露公网。 |
| AI 搜索原型 | Pagefind + AI Gateway + Workers AI / AI Search | 先用静态搜索；AI 调用必须有缓存、限流和日志。 |
| 实时小应用 | Durable Objects + 连接休眠 + Queues | 房间状态、后台任务和重试都能先小规模验证。 |

## 免费额度决定架构

先把项目拆成几层，再把每一层放到最合适的免费额度里。这样写出来的架构更稳，也更容易知道哪一天该升级。

| 架构层 | 最先看的免费额度 | 起步做法 |
| --- | --- | --- |
| 静态内容 | 静态资产请求免费且不限量；Pages Free 每月 500 次构建。 | 文档、官网、博客、前端资源先走 Workers Static Assets / Pages。 |
| 动态接口 | Workers Free 100,000 次请求/天，10 ms CPU/次调用。 | 只让接口、第三方回调、评论、表单和后台入口进 Worker。 |
| 关系数据 | D1 Free 5M 行读取/天、100k 行写入/天、5 GB 总存储。 | 评论、反馈、小后台先用 D1；常查字段先建索引。 |
| 配置缓存 | KV Free 每天 100k 读取、1k 写入、1 GB 存储。 | 放配置、开关、低频缓存；写多和强一致状态不要放 KV。 |
| 文件对象 | R2 Standard 每月 10 GB、1M 写入类操作、10M 读取类操作，出站带宽免费。 | 附件、图片原图、导出物放 R2；公开下载配缓存。 |
| 异步任务 | Queues Free 10,000 次操作/天。 | 邮件、通知、导入、后处理进队列；重试和消息大小要计入操作数。 |
| 强一致状态 | Durable Objects Free 100,000 次请求/天、13,000 GB-s/天。 | 房间、会话、限流器用 DO；实时连接优先做休眠。 |
| 公开写入口 | Turnstile Free 20 个小组件，每个小组件 10 个域名。 | 登录、评论、表单、上传和 AI 搜索入口先加防刷边界。 |
| AI 与搜索 | Pagefind 不打后端；Workers AI 每天 10,000 Neurons；AI Search 新实例每月 20,000 次查询。 | 静态搜索先 Pagefind；自然语言搜索验证后再接 AI Gateway / Workers AI / AI Search。 |

## 免费阶段最容易撞的红线

免费额度最怕的不是数字小，而是入口放错。先把这些红线写进项目设计里，后面就少很多账单和排障焦虑。

| 红线 | 为什么危险 | 起步做法 |
| --- | --- | --- |
| 静态资源误进 Worker | 图片、脚本、样式和 HTML 会消耗动态请求与 CPU。 | 静态内容先放 Workers Static Assets / Pages，只让接口进 Worker。 |
| 公开写入口裸奔 | 评论、表单、上传、搜索和第三方回调会被刷。 | Turnstile、限流、鉴权和业务配额先上。 |
| D1 无索引查询 | D1 按读取行数计量，全表扫描会浪费免费额度。 | 常查字段建索引，列表查询分页，后台报表离线跑。 |
| KV 写多或强一致 | KV 免费写入少，且不是强一致数据库。 | 配置和低频缓存用 KV；状态、评论、订单用 D1 或 Durable Objects。 |
| R2 热点读取无缓存 | R2 免出站带宽费，但读取类操作仍然计量。 | 公开下载配 CDN 缓存，私有下载用签名链接。 |
| AI 无缓存无上限 | Workers AI、AI Gateway、AI Search 和 Vectorize 都会被开放式调用放大。 | Pagefind 先顶上；AI 入口加缓存、限流、模型选择和日志。 |

<Aside type="caution" title="先记住这条">
免费额度先服务架构顺序：静态内容 -> 静态层；文件 -> R2；关系数据 -> D1；配置缓存 -> KV；强一致状态 -> Durable Objects；耗时任务 -> Queues；公开写入口 -> Turnstile。
</Aside>

## 什么时候付 5 美元/月

Workers Paid 是账号级订阅，不是 Cloudflare Pro，也不是账单封顶。它最适合在 Free 已经证明项目有人用之后再开。

| 该付的信号 | 为什么 |
| --- | --- |
| Worker 请求稳定接近 100,000 次/天。 | Free 会在日额度耗尽后出问题；Paid 改成月度已包含用量。 |
| CPU 经常超过 10 ms。 | 服务端渲染、AI 前处理、批处理和大内容解析需要更高 CPU 上限。 |
| 日志 3 天不够排障。 | Paid Workers Logs 留存和月度额度更适合生产。 |
| D1、KV、Queues、Durable Objects 已经是核心功能。 | 这些产品在 Paid 下进入更高月度额度和付费扩展。 |
| 需要更多定时任务、子请求、Worker 数量或更大构建包。 | 这是开发者平台工程化能力，不是单次流量问题。 |

| 不该误会 | 正确理解 |
| --- | --- |
| 5 美元/月不是 Cloudflare Pro。 | 它不增加 WAF、缓存规则、Bot、证书或域名计划权益。 |
| Static Assets 不是所有请求免费。 | 命中静态资产才免费；服务端渲染、动态函数、接口、整站先跑 Worker 会走 Workers 计费。 |
| R2 出站带宽免费不是 R2 全免费。 | 存储量、写入类操作和读取类操作仍然要看。 |
| 预算提醒不是硬封顶。 | 它只发邮件，不会暂停服务或阻止继续计费。 |
| AI 免费额度不是公开无限用。 | 必须做缓存、限流、模型选择和日志观察。 |

## 这是什么

Cloudflare Playbook 不是官方文档的搬运，而是一份面向真实项目的实践索引。它会把 Cloudflare 的产品能力翻译成普通开发者能使用的判断框架：免费额度能撑到哪里、什么时候用、怎么组合、怎么部署、怎么验证、有什么风险。

Cloudflare 覆盖了计算、存储、网络、安全、AI 和可观测性等基础设施能力。对独立开发者和早期团队来说，理解这些产品的边界和组合方式，可以显著降低项目启动和运维成本。这份知识库的目标，是把这些能力整理成一张能看懂、能落地、能持续更新的大图谱。

## 先看免费额度

最后核对日期：2026-06-17。Cloudflare 的额度和价格变化很快，下面这张表只做入口判断；写预算、报价或上线方案时，请回到 [免费与付费边界](/platform/free-paid/) 和 [Billing](/platform/billing/) 继续核对官方 pricing / limits。

多数人先关心的不是产品名，而是一个项目能不能先免费跑起来。本站把 Cloudflare 的免费额度当作主线来读：先确认静态流量、动态请求、数据库、对象存储、安全和搜索的免费边界，再判断什么时候值得开 `$5/month` 的 Workers Paid 或其他付费产品。

四个结论先放前面：

1. 静态站、文档站、官网和博客优先放在 Workers Static Assets 或 Pages，静态资产请求免费且不限量。
2. 小 API、评论、Webhook、表单、D1、KV、R2、Queues、Durable Objects、Turnstile、Access、Workers AI、AI Search 都有可用的免费起步空间。
3. `$5/month` 的 Workers Paid 是动态能力和工程化能力的第一道付费台阶，不是 Cloudflare Pro，也不是预算封顶。
4. 免费额度不是消费封顶。Budget alerts 只发提醒，真正控制成本要靠静态优先、缓存、限流、队列、配额和权限。

## 免费阶段能怎么玩

这份 Playbook 的默认路线很简单：先让项目站在免费额度里跑通，再为明确的瓶颈付费。能静态就静态，能缓存就缓存，能异步就异步，真正需要计算、写入、AI 或浏览器自动化的请求再进入对应产品。

| 你想做什么 | 免费阶段优先组合 | 为什么划算 |
| --- | --- | --- |
| 文档站、官网、博客 | Workers Static Assets / Pages + CDN + Pagefind + Web Analytics | 主流量是静态资产，搜索在浏览器本地完成。 |
| 小 API、Webhook、评论 | Workers Free + D1 Free + Turnstile + WAF 最小规则 | 动态请求和写入量低时，可以先验证真实需求。 |
| 文件上传和下载 | R2 Standard Free + presigned URL + CDN cache | 不收 R2 egress bandwidth charge，重点盯 storage 和 operations。 |
| 后台、预览环境、内网工具 | Zero Trust Free + Access + Tunnel | 50 users 内能先把后台从公网裸露状态收回来。 |
| 搜索和 AI 试验 | Pagefind + Workers AI 免费 Neurons + AI Gateway + AI Search Free | 先把索引和调用链路跑通，再看自然语言搜索是否值得付费。 |
| 异步任务和实时状态 | Queues Free + Durable Objects Free + WebSocket hibernation | 后台任务、房间、限流器先小规模跑，等成为核心路径再升级。 |

## 什么时候付 $5 Workers Paid

不要为了“看起来专业”提前付费。下面这些信号出现时，再认真考虑 Workers Paid：

| 信号 | 为什么可能要付费 |
| --- | --- |
| Worker 接近 100,000 requests/day | Paid 改成月度 included usage，并允许继续按量扩展。 |
| 单次 CPU 经常超过 10 ms | Paid 有更高单次 CPU 上限和月度 CPU included usage。 |
| 生产排障需要更久日志 | Paid 的 Workers Logs、Trace Events Logpush 和观测能力更适合线上服务。 |
| D1、KV、Queues、Durable Objects 已经是正式链路 | Paid 把这些开发者平台产品放进更高的月度额度和付费模型里。 |
| 需要 Containers、任意收件人 Email Sending、Sandbox SDK 等能力 | 这些通常不是免费文档站需求，而是业务进入下一阶段的信号。 |

## 免费额度大全（首页版）

| 能力 | Free 重点额度 | 付费信号 |
| --- | --- | --- |
| DNS | Free / Pro / Business 不对 DNS queries 收费；新 Free zone 为 200 records/zone，旧 Free zone 为 1,000。 | 记录数、DNS SLA、企业治理和专属支持成为瓶颈时看 Enterprise。 |
| CDN / Cache | CDN 可用于所有计划；Free 有 10 条 Cache Rules，最大可缓存文件 512 MB。 | 需要更多 Cache Rules、更短 Edge TTL、复杂 purge 或 Cache Reserve 时单独评估。 |
| SSL/TLS | Universal SSL、Origin CA、Always Use HTTPS、HSTS、TLS 1.3、Authenticated Origin Pulls 可在 Free 使用。 | 需要 Advanced Certificate Manager、Custom Certificates、Keyless SSL 时看更高计划或 add-on。 |
| DDoS | 所有计划都有 standard, unmetered DDoS protection，HTTP DDoS 和 Network-layer DDoS 都覆盖。 | 需要高级 DDoS overrides、Log action、企业告警和网络层合同能力时升级。 |
| WAF / Rate Limiting | WAF available on all plans；Free 有 5 条 Custom Rules、1 条 Rate Limiting Rule、Free Managed Ruleset。 | 登录、后台、API、上传、评论的规则不够用，或需要更强 Bot / API 防护时升级。 |
| Turnstile | Free 最多 20 widgets，每个 widget 10 hostnames；挑战和验证请求不限量。 | widget、hostname、analytics lookback 或企业策略不够时看 Enterprise。 |
| Zero Trust / Access / Tunnel | Zero Trust Free 为 $0 forever，50 user limit；Tunnel 可发布 public hostname。 | 用户数、日志留存、设备姿态、Gateway、DLP 或企业审计要求上来后升级。 |
| Web Analytics | 免费、隐私优先；代理站点不限，非代理站点 10 个；Free 计划 Rules limit 为 0。 | 需要 Web Analytics rules、复杂归因或外部数据管道时再扩展。 |
| 静态资产请求 | Workers Static Assets / Pages 静态资产请求免费且不限量。 | 文件数、单文件大小、构建并发或动态 Functions 成为瓶颈时再升级。 |
| Pages | 500 builds/month、1 concurrent build、100 custom domains/project、20,000 files/site、25 MiB/file。 | 构建次数、并发构建、文件数或 Pages Functions 成本增长时升级。 |
| Workers 动态请求 | 100,000 requests/day，10 ms CPU/invocation，UTC 0 点重置。 | 接近每日请求上限、CPU 超过 10 ms、需要更长任务或生产日志时看 Workers Paid。 |
| Workers Paid | Free 不需要固定月费；静态资产请求仍免费。 | 每账号每月最低 `$5`，包含 10M requests/month、30M CPU ms/month 和更高平台限制。 |
| D1 | 5M rows read/day、100k rows written/day、5 GB total storage。 | 读写量稳定增长、需要移除每日硬限制、更大数据库或生产 SLA 时升级。 |
| KV | 100k reads/day、1k writes/day、1k deletes/day、1k lists/day、1 GB storage。 | 写入、删除、列表操作变多，或读量进入月度包和按量扩展时升级。 |
| R2 Standard | 10 GB-month/month、1M Class A/month、10M Class B/month；egress 免费。 | 存储、上传、读取次数超过免费包；冷数据 Infrequent Access 另算。 |
| Queues | 10,000 operations/day，消息保留 24 小时。 | 后台任务稳定、需要更长保留期、更多 operations 或 Dead Letter Queue 时升级。 |
| Durable Objects | 可用 SQLite-backed DO；100,000 requests/day、13,000 GB-s/day。 | 房间、会话、限流器、WebSocket 状态成为核心路径后升级，并使用 hibernation。 |
| Workers Logs | 200,000 log events/day，保留 3 天。 | 生产排障需要 7 天日志、Trace Events Logpush 或外部日志目的地时升级。 |
| Analytics Engine | 100,000 data points/day、10,000 read queries/day。 | 高基数指标成为产品能力，或报表查询进入月度规模时升级。 |
| Workers AI | Free / Paid 都有 10,000 Neurons/day 免费分配。 | 超过每日 Neurons 需要 Workers Paid，之后按 Neurons 计费。 |
| AI Gateway | 核心功能免费，包含 dashboard analytics、caching、rate limiting；Free 持久日志 100,000 logs total。 | 需要 10M logs/gateway、Logpush、长期 AI 调用审计时升级。 |
| AI Search | 新实例 open beta；Free 有 100 instances/account、100,000 files/instance、20,000 queries/month、500 crawled pages/day。 | 自然语言搜索成为核心体验、查询量增长或需要 unlimited crawling 时升级。 |
| Vectorize | 30M queried vector dimensions/month、5M stored vector dimensions。 | 语义检索规模超过免费维度，或 RAG 成为正式产品能力时升级。 |
| Images | Free 每月 5,000 unique transformations；超过后新 transformation 返回错误，不直接收费。 | 需要更多图片转换、Images storage 或 Images delivery 时购买 Images Paid。 |
| Browser Run | Free 有 10 minutes/day browser hours，Browser Sessions 3 concurrent browsers。 | 截图、PDF、网页抓取等浏览器自动化稳定运行后看 Workers Paid。 |
| Zaraz | 每个 account 每月 1,000,000 free Zaraz Events。 | 第三方脚本治理流量超过免费包，或需要持续启用 paid events 时升级。 |
| Budget alerts | Pay-as-you-go 账号可按美元阈值发邮件提醒 usage-based spend。 | 它不是硬封顶；需要在业务侧做限流、配额、队列和权限。 |

第一条成本铁律：**主流量尽量停在静态资产、CDN、R2 和缓存层，只让真正需要计算的请求进入 Worker。**

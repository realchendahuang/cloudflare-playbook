## 这是什么

Cloudflare Playbook 不是官方文档的搬运，而是一份面向真实项目的实践索引。它会把 Cloudflare 的产品能力翻译成普通开发者能使用的判断框架：什么时候用、免费边界在哪里、怎么组合、怎么部署、怎么验证、有什么风险。

Cloudflare 覆盖了计算、存储、网络、安全、AI 和可观测性等基础设施能力。对独立开发者和早期团队来说，理解这些产品的边界和组合方式，可以显著降低项目启动和运维成本。这份知识库的目标，是把这些能力整理成一张能看懂、能落地、能持续更新的大图谱。

## 推荐阅读顺序

1. 先读 [学习路线](/start/)，知道这份文档站怎么用。
2. 再读 [Fundamentals](/platform/fundamentals/)，先搞清楚 Account、Zone、代理状态和源站边界。
3. 然后读 [产品地图](/platform/)，建立 Cloudflare 产品之间的关系。
4. 再进入 [架构模式](/architecture/)，把产品组合成可落地方案。
5. 最后进入 [实战案例](/recipes/)，按一个具体场景跑通。

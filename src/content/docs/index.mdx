## 先别看产品，先看免费额度

最后核对日期：2026-06-18。

Cloudflare 的价格、额度和计划边界会变化。本站数字以官方价格和限制页面为最终依据；首页先把起步项目最该看的免费额度摊开，完整口径见 [免费额度大全](/platform/free-paid/)。

这个 Playbook 不从产品名开始，而是从免费额度开始。免费额度不是附录，它决定一个普通项目最初怎么设计。读者真正关心的是四件事：

- 项目能不能先 0 元跑起来。
- 0 元能跑到什么程度。
- 跑到哪里才该付费。
- 付的那 5 美元到底买到什么。

| 先问 | 直接答案 |
| --- | --- |
| 文档站、官网、博客能不能 0 元跑？ | 可以。静态资产请求免费且不限量，先用 Workers Static Assets / Pages。 |
| 小 API、评论、表单能不能 0 元验证？ | 可以。Workers Free、D1、KV、R2、Queues、Durable Objects 都有免费层。 |
| 免费阶段最该防什么？ | 防公开写入口被刷、防静态流量误进 Worker、防数据库全表扫描、防 AI 调用不设限。 |
| 什么时候付 5 USD/month？ | 动态请求、CPU、日志、队列、D1/KV/DO 等开发者平台能力进入真实生产后再付。 |

## 免费额度怎么读

不是“看到免费就全都用”，而是把项目拆成几条线：静态内容能不能停在静态层，动态入口能不能压到最少，数据和文件有没有放对地方，公开写入口有没有防刷，AI 和日志有没有上限。

| 先判断 | 免费层最友好的做法 |
| --- | --- |
| 主流量是什么 | 文档、官网、博客和前端资源先走静态资产层。 |
| 哪些请求必须运行代码 | 只让 API、Webhook、评论、表单和后台接口进入 Worker。 |
| 数据是什么形态 | 关系数据进 D1，文件进 R2，读多写少配置进 KV，强一致状态进 Durable Objects。 |
| 谁能公开写入 | 登录、评论、表单、上传、搜索和 Webhook 先加 Turnstile、限流或业务权限。 |
| 哪些会放大账单 | AI、日志、浏览器自动化、媒体和大文件下载都要先设边界。 |

## All the fun：0 元能跑的项目，不止静态站

免费额度最有价值的地方，是能把真实项目先组合出来。不要先背产品名，先把项目拆成静态流量、动态入口、数据、文件、安全、日志和搜索。

| 项目形态 | 0 元起步核心 | 第一条红线 |
| --- | --- | --- |
| 开源文档站 / 官网 | 静态资产层或 Pages + Pagefind + Web Analytics | 构建次数、文件数、静态请求误进 Worker。 |
| 教程站轻社区 | 静态资产层 + Twikoo Cloudflare + D1 + Turnstile | D1 写入、公开入口刷量、日志留存。 |
| 小 API / Webhook | Workers Free + D1 / KV + WAF / Turnstile | 100,000 次动态请求/天、10 ms CPU/次调用。 |
| 上传下载工具 | R2 + 签名链接 + CDN 缓存 | 每月 10 GB 存储量、写入类和读取类操作。 |
| 小团队后台 | Access + Tunnel + D1 | 50 users、审计日志、后台权限。 |
| AI 搜索原型 | Pagefind + AI Gateway + Workers AI / AI Search | 模型用量、搜索次数、网关日志。 |

| 每月 3000 万查询向量维度 | Vectorize Free 查询维度额度。 | 小规模语义搜索可试验；先整理文档再做向量化。 |

</Aside>

## Cloudflare 免费额度大全

这张速查表先回答“免费能不能跑”和“5 美元有没有帮助”。完整产品级表格、计费解释和官方来源见 [免费额度大全](/platform/free-paid/)。

| 能力 | Free 额度 / 免费边界 | 5 USD/month Workers Paid 是否相关 | 最容易撞墙 |
| --- | --- | --- | --- |
| DNS | Free / Pro / Business 不按 DNS 查询收费；新 Free zone 每个域名 200 条记录，旧 Free zone 1,000 条。 | 无关。DNS 属于域名和账号基础能力。 | 记录数、团队权限、企业级 DNS 需求。 |
| CDN / Cache | 所有计划可用；Free 有 10 条缓存规则；Free / Pro / Business 最大可缓存文件 512 MB。 | 无关。Workers Paid 不增加缓存规则。 | 复杂缓存规则、大文件、频繁清缓存。 |
| SSL/TLS | Universal SSL、Origin CA、Always Use HTTPS、HSTS、TLS 1.3 可在 Free 使用。 | 无关。证书高级能力看域名计划或单独付费能力。 | 自定义证书、Keyless SSL、复杂证书治理。 |
| DDoS Protection | 所有计划都有标准且不限量的 DDoS 防护。 | 无关。高级 DDoS 配置看企业能力或单独付费能力。 | 源站 IP 暴露、应用层恶意写入、误伤排查。 |
| WAF 自定义规则 | Free 5 条；Pro 20；Business 100。 | 无关。Workers Paid 不增加 WAF 规则。 | 规则数量、事件历史、Bot / API 高级防护。 |
| 限流规则 | Free 1 条；Pro 2；Business 5。 | 无关。它跟 WAF / 域名计划走。 | 登录、评论、上传、Webhook 入口太多。 |
| Turnstile | Free 每账号 20 个小组件，每个小组件 10 个域名，挑战和验证请求不限量。 | 无关。Enterprise 才提升小组件、域名和统计边界。 | 多域名、多租户、品牌和高级 Bot 检测。 |
| Zero Trust / Access / Tunnel | Free 适合 50 个用户以内的小团队；Tunnel 可发布公开主机名。 | 无关。Zero Trust 用户席位和日志另看计划。 | 用户数、网关、数据防泄漏、设备姿态、审计日志。 |
| Web Analytics | 免费；代理站点不限，非代理站点 10 个。 | 无关。 | 复杂归因、长期原始日志、跨系统分析。 |
| Workers Static Assets | 静态资产请求免费且不限量；存储 Assets 不额外收费。 | 相关但不收费。Paid 后静态请求仍免费且不限量。 | 整站先跑 Worker、SSR、API 路径误打进 Worker。 |
| Pages | Free 每月 500 次构建、1 个并发构建、每项目 100 个自定义域名、每站 20,000 个文件、单文件 25 MiB。 | Pages Functions 按 Workers 计费。纯静态不需要先付。 | 构建次数、文件数、Functions 请求。 |
| Workers 请求 | Free 100,000 次/天，UTC 0 点重置。 | Paid 含每月 1000 万次请求，超出 0.30 USD/million。 | 公开 API、评论、搜索代理、SSR 动态请求。 |

## 什么时候付 5 USD/month

Workers Paid 是账号级订阅，不是 Cloudflare Pro，也不是账单封顶。它最适合在 Free 已经证明项目有人用之后再开。

| 该付的信号 | 为什么 |
| --- | --- |
| Worker 请求稳定接近 100,000 次/天。 | Free 会在日额度耗尽后出问题；Paid 改成月度已包含用量。 |
| CPU 经常超过 10 ms。 | SSR、AI 前处理、批处理和大 payload 解析需要更高 CPU 上限。 |
| 日志 3 天不够排障。 | Paid Workers Logs 留存和月度额度更适合生产。 |
| D1、KV、Queues、Durable Objects 已经是核心功能。 | 这些产品在 Paid 下进入更高月度额度和付费扩展。 |
| 需要更多定时任务、子请求、Worker 数量或更大构建包。 | 这是开发者平台工程化能力，不是单次流量问题。 |

| 不该误会 | 正确理解 |
| --- | --- |
| 5 USD/month 不是 Cloudflare Pro。 | 它不增加 WAF、缓存规则、Bot、证书或域名计划权益。 |
| Static Assets 不是所有请求免费。 | 命中静态资产才免费；SSR、Functions、API、整站先跑 Worker 会走 Workers 计费。 |
| R2 出站带宽免费不是 R2 全免费。 | 存储量、写入类操作和读取类操作仍然要看。 |
| Budget alerts 不是硬封顶。 | 它只发邮件，不会暂停服务或阻止继续计费。 |
| AI 免费额度不是公开无限用。 | 必须做缓存、限流、模型选择和日志观察。 |

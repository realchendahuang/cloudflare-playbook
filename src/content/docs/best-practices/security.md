# 安全边界

安全边界先从入口、凭据、写操作和生产资源开始。

## 上线前检查

- 域名是否只暴露需要公开的入口。
- 管理后台是否使用 Access 或其他鉴权保护。
- API 是否校验身份、来源和权限。
- Webhook 是否校验签名。
- R2、D1、KV、Queues 等资源是否只通过绑定访问。
- 密钥是否使用 Cloudflare Secrets 或平台环境变量管理。

## 常见风险

- 把服务端密钥写进前端构建产物。
- 公开了本该内部访问的调试接口。
- 允许匿名用户触发高成本 AI 或文件操作。
- 缺少回滚策略就直接修改生产数据。

## 写案例时必须说明

如果案例涉及写数据库、发邮件、上传文件、调用 AI 或修改生产配置，必须写明权限边界和失败处理方式。

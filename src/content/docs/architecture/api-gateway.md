# API 网关

API 网关适合把多个后端、第三方 API 或内部服务包装成一个统一入口。

## 架构

```text
客户端
  │
  ▼
Worker
  ├─ 校验身份
  ├─ 读取 KV 配置
  ├─ 查询 D1
  ├─ 写入 Queue
  └─ 转发源站或第三方 API
```

## 常见能力

- 鉴权：校验 token、签名、来源和权限。
- 限流：用 KV 或 Durable Objects 管理访问频率。
- 缓存：缓存公开接口或低频变化数据。
- 观测：记录请求状态、上游延迟和错误分类。

## 判断标准

如果逻辑可以在一次请求里完成，并且状态需求不复杂，Workers 通常足够。如果需要强一致对象状态，再引入 Durable Objects。

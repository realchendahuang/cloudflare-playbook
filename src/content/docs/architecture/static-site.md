---
title: 静态内容站
---

## 一句话判断

| 场景 | 默认选择 |
| --- | --- |
| 纯文档站、官网、博客 | Pages 或 Workers Static Assets。 |
| 文档站 + 接口 / 评论 / 表单 | Workers Static Assets + 动态接口。 |
| 大量图片、附件、导出物 | R2 + 缓存策略。 |
| 普通站内搜索 | Pagefind。 |
| 私有后台或预览环境 | Access / Tunnel。 |

## 部署入口

| 入口 | 适合 | 判断 |
| --- | --- | --- |
| Workers Static Assets | 文档站旁边还会有接口、D1、R2、Queues、AI Gateway 或评论服务。 | Worker 和静态资产同一部署模型，长期更适合 Cloudflare-first 项目。 |
| Pages | 纯静态文档、官网、博客、营销页、PR 预览优先的项目。 | Git 工作流轻，静态资产请求同样免费；动态函数按 Workers 计费。 |

## 路径处理

| 路径 | 推荐处理方式 | 成本和风险 |
| --- | --- | --- |
| `/`、`/docs/*`、`/assets/*` | 静态资产直接返回。 | 静态资产请求免费且不限量，不交给 Worker 脚本。 |
| `/pagefind/*` | 构建期搜索索引。 | 用户搜索在浏览器本地完成，不消耗后端请求。 |
| 动态接口 | Worker。 | 按 Workers 请求和 CPU 计费，需要日志、限流和错误处理。 |
| `/files/*` | Worker + R2。 | Worker 做权限和签名，文件本体进 R2。 |
| `/admin/*` | Worker + Access / Tunnel。 | Access 校验通过后进入后台。 |

## 成本项

| 成本项 | 实践判断 |
| --- | --- |
| 静态资产请求 | 文档、官网、博客的主流量停在 Workers Static Assets / Pages。 |
| Worker 动态请求 | 评论、表单、第三方回调、小接口可以先跑；公开写入口要限流。 |
| 构建产物 | 站点构建包只放页面和前端资源；大量图片、附件、导出物进 R2。 |
| R2 | R2 仍然计算存储和操作，公开下载热点要看读取操作。 |
| D1 | 评论、表单、小型后台；高频分析和大表扫描走分析或数据管道。 |

## 上线前只查这几件事

| 检查 | 判断 |
| --- | --- |
| 静态页面 | 首页、文章页、404、CSS / JS / 图片都能访问。 |
| 搜索 | 搜索索引存在，线上搜索能搜到文章。 |
| 动态路径 | 只有接口、评论、上传、后台等必要路径交给 Worker。 |
| 文件上传 | Worker 生成签名或受控入口，R2 存储桶不开放任意写入。 |
| 管理后台 | Access / Tunnel 先挡在前面，再做业务权限。 |
| 日志 | 能定位问题，不记录密钥、登录凭证和正文隐私。 |
